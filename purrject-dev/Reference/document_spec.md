# Document Specification

Module `document` files must follow this format:

## Required Content (and only these)

1. **Language** — The programming language used by the module.
2. **API** — Public interfaces exposed for external callers.
3. **API Description** — Brief explanation of each API. Omit if the name is self-explanatory.

## Style

- Concise. No prose, no background, no usage examples.
- Use a flat structure: language as a heading, API list below.
